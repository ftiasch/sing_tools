# mt3000-bj OpenWrt 旁路由

2026-06-21 从 `camp-bb` (10.11.1.2) 迁移 dnsmasq + sing-box 透明代理到 GL-MT3000。

## 硬件

- 型号: GL.iNet GL-MT3000
- SoC: MediaTek MT7981 (aarch64_cortex-a53)
- 内存: 490MB
- 系统: ImmortalWrt 24.10.6

## 网络拓扑

```
WAN (上游)                         LAN (本机)
10.11.1.1                          10.11.1.0/24
    │                                  │
    └──── br-lan (10.11.1.3/24) ───────┘
              │
         sing-box TUN (172.16.0.1/30)
              │
         tailscale0 → mt3000-st (vless relay)
```

单臂桥接，eth0+eth1+Wi-Fi 全在 br-lan。IP `10.11.1.3/24`，gateway `10.11.1.1`。

## 关键配置

### network

```
lan.proto=static, ip=10.11.1.3/24, gateway=10.11.1.1
bridge: eth0 + eth1 + phy0-ap0 + phy1-ap0
no WAN interface (单臂)
```

### dhcp

```
dnsmasq port=0 (不做递归 DNS)
DHCP: 10.11.1.82-244, gateway=10.11.1.3, DNS=223.5.5.5
DHCPv6 disabled, RA disabled
```

### firewall

```
lan zone: input/output/forward ACCEPT, device=tun0
tailscale zone: ACCEPT, device=tailscale0
无 WAN zone (单臂不需要)
```

### sing-box

- 版本: 1.13.13 (musl, 手动升级)
- 配置由仓库 `templates/base.yaml` + `templates/mt3000-bj.yaml` 生成
- TUN: `172.16.0.1/30`, auto_route=true, strict_route=true
- 出站: vless relay 到 mt3000-st (100.116.175.64:48443) via tailscale0
- DNS 劫持: `223.5.5.5` → sing-box DNS
- Dashboard: 官方 sing-box dashboard (http://10.11.1.3:9090)

### 已停服务

```
wpad (hostapd + wpa_supplicant) — Wi-Fi 不承载流量
odhcpd — IPv6 已禁用
uhttpd — 不需要 LuCI
```

## 部署

```bash
# 生成配置
uv run app.py generate mt3000-bj

# 部署
uv run pyinfra inventory.py deploy.py --limit mt3000-bj -y
```

## 排障记录

1. **fw4 drop TUN 流量**: `tun0` 必须加入 lan zone (`uci add_list firewall.@zone[0].device=tun0`)
2. **sing-box musl**: GL-MT3000 用 musl libc，需要 `linux-arm64-musl.tar.gz`
3. **resolv.conf**: 路由器本身 DNS 指向 `223.5.5.5`，由 sing-box hijack
4. **icmp local route 刷屏**: TUN `auto_route` 导致，暂不处理
